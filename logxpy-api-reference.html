üöÄ 4-Step Quick Start Tutorial

1

Initialize the Logger

Auto-generate log filename from your script or specify one explicitly. Async logging is enabled by default for high performance.

from logxpy import log # Auto-generate: script.py ‚Üí script.log log.init(clean=True) # Or specify explicitly log.init("app.log", level="DEBUG", mode="w") # Force sync mode if needed (async is default) log.init("app.log", async_enabled=False)
2

Log Messages with Fluent API

All methods return self for chaining. Use structured logging with key-value pairs.

# Basic logging log.info("User logged in", user_id=123, username="alice") log.success("Operation completed", duration=0.145) # Fluent chaining log.debug("Starting").info("Processing").success("Done") # Colored output for CLI viewer log.set_foreground("cyan").info("Cyan text").reset_foreground() # Checkpoint with emoji prefix log.checkpoint("Step 1 complete")
3

Use Actions for Nested Operations

Track hierarchical operations with automatic start/success/failed status and duration tracking.

from logxpy import start_action, start_task # Top-level task with start_task("process_order", order_id="ORD-123") as task: log.info("Validating order") # Nested action with start_action("db:query", table="orders"): log.info("Fetching from database") # ... database operation ... # Another nested action with start_action("http:request", url="/api/notify"): log.info("Sending notification") # ... HTTP request ... log.success("Order processed")
4

View and Analyze Logs

Use the CLI viewer to render beautiful colored trees or the parser for programmatic analysis.

# View as colored tree $ logxpy-view app.log # Filter and analyze $ logxpy-view app.log --status failed --min-duration 1.0 # Live monitoring $ logxpy-view tail app.log --dashboard # Programmatic analysis from logxy_log_parser import parse_log logs = parse_log("app.log") errors = logs.filter.by_level("error").by_time_range("2024-01-01", "2024-12-31")

üöÄ 4-Step CLI Viewer Tutorial

1

Install and Basic View

Install the CLI viewer and view your logs as a beautiful colored tree.

# Install $ pip install logxpy-cli-view # Basic tree view $ logxpy-view app.log # One-line format $ logxpy-view app.log --format oneline
2

Filter Your Logs

Use powerful filters to find exactly what you need.

# Filter by status $ logxpy-view app.log --status failed # Filter by action type $ logxpy-view app.log --action-type "db:*" # Filter by duration $ logxpy-view app.log --min-duration 1.0 # Filter by task UUID (Sqid) $ logxpy-view app.log --task-uuid Xa.1
3

Advanced Queries

Use JMESPath for complex queries and date ranges.

# JMESPath query $ logxpy-view app.log --select "[?status=='failed']" # Date range $ logxpy-view app.log --start 2024-01-01 --end 2024-12-31 # Search keyword $ logxpy-view app.log --keyword "error" # Limit nesting depth $ logxpy-view app.log --max-level 3
4

Export and Monitor

Export to various formats and monitor logs in real-time.

# Export to different formats $ logxpy-view export app.log -f json -o output.json $ logxpy-view export app.log -f csv -o output.csv $ logxpy-view export app.log -f html -o output.html # Live monitoring $ logxpy-view tail app.log $ logxpy-view tail app.log --dashboard # Statistics $ logxpy-view stats app.log

1

Parse Log Files

Parse log files with simple one-liners or use the full API for more control.

from logxy_log_parser import parse_log, parse_line, analyze_log # Parse entire file logs = parse_log("app.log") # Parse single line entry = parse_line('{"ts": 123, "msg": "test"}') # Full analysis report report = analyze_log("app.log") print(report.summary)
2

Filter and Query

Use chainable filters to find exactly what you need.

from logxy_log_parser import LogFilter # Chainable filters errors = LogFilter(logs).by_level("error").by_time_range("2024-01-01", "2024-12-31") # Filter by task by_task = logs.filter.by_task_uuid("Xa.1") # Filter by action type db_ops = logs.filter.by_action_type("db:*") # Slow actions slow = logs.filter.slow_actions(threshold=5.0)
3

Analyze Performance

Use LogAnalyzer to get insights about your application's performance.

from logxy_log_parser import LogAnalyzer analyzer = LogAnalyzer(logs) # Slowest actions slowest = analyzer.slowest_actions(10) # Error summary errors = analyzer.error_summary() # Duration by action type durations = analyzer.duration_by_action() # Failure rates failure_rates = analyzer.failure_rate_by_action()
4

# Export to different formats logs.to_json("output.json") logs.to_csv("output.csv") logs.to_html("output.html") logs.to_markdown("output.md") # Real-time monitoring from logxy_log_parser import LogFile logfile = LogFile.open("app.log") for entry in logfile.watch(): print(entry) # Wait for specific message entry = logfile.wait_for_message("ready", timeout=30)
