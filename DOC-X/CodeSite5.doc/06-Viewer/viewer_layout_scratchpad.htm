CodeSite Live/File Viewer

Layout: Scratch Pad

    

To display the Scratch Pad, use the Scratch Pad command.

The Scratch Pad is designed to display non-persistent information.  The Scratch Pad is very effective when you want to keep track of some information but you do not need to record the information in the message log.  Consider tracking the mouse position during mouse move events.  It is pointless to send this information to the message log because too much data results.  Furthermore, the only relevant value is the most recent mouse position.

Scratch Pad messages are sent to the Live Viewer by using the T|CodeSiteLogger.Write method.  Scratch Pad messages are sent to the Live Viewer like normal messages, but have a ScratchPad message type that directs the viewer to write the message text to the appropriate Scratch Pad line.  Each line is identified by a Line ID which appears at the left of each display line in the Scratch Pad.  If a new Scratch Pad message is received and a corresponding Line ID does not exist, a new line is added to the Scratch Pad.   If a Line ID does exist, the existing value is overwritten with the new value.

Each time the data in a Scratch Pad line is updated, the light to the far left of each line is flashed.  To change the color of the light, use the  Options command.

Please note that information written to the Scratch Pad is not stored when the information in the Message Log is saved to a file. In addition, Scratch Pad messages are not dispatched to the CodeSite Log Files, only the CodeSite Live Viewer.
