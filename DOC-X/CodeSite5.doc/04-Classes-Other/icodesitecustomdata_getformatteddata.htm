Formats the custom data.

Frameworks

.NET

*Not Available in CodeSite Express

Syntax

Delphi

Remarks

GetFormattedData is where you will actually format your custom data as desired.  Specifically, developers will populate the Data parameter, which is a TStringList in Delphi/C++Builder, and a StringBuilder in .NET.  If InspectorType is set to StockText (itStockText in Delphi/C++Builder; CodeSiteInspectorType.StockText in .NET), then simply add lines to the Data parameter.

If the InspectorType is set to StockProperties (itStockProperties in Delphi; CodeSiteInspectorType.StockProperties in .NET), add a line in the form: <name>=<value> to the Data parameter, where <name> is the name of the property, and <value> is the current value for that property.

If the InsepctorType is set to StockTable (itStockTable in Delphi; CodeSiteInspectorType.StockTable in .NET), then each line added to the Data parameter will represent a row. Columns are defined by adding Tab (#9 or \t) characters within the string at the column boundaries.

If the InsepctorType is set to StockHierarchy (itStockHierarchy in Delphi; CodeSiteInspectorType.StockHierarchy in .NET), then each line added to the Data parameter will represent a node in the hierarchy. The hierarchical structure is determined by the number of tab (#9 or \t) characters that appear at the beginning of each line.

When the InspectorType is set to Custom (itCustom in Delphi; CodeSiteInspectorType.Custom in .NET), then you decide how you want to represent the data in the Data string list.  Setting InspectorType to Custom will also require a custom inspector being created.
