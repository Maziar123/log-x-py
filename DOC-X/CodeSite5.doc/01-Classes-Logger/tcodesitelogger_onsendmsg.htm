TCodeSiteLogger.OnSendMsg Event

    

Occurs just before a message is sent to the current destination.

Frameworks

Delphi

.NET

*Not Available in CodeSite Express

Syntax

Delphi

property OnSendMsg: TCodeSiteSendMsgEvent

 

TCodeSiteSendMsgEvent = procedure ( Sender: TObject; MsgData: TCodeSiteMsgData; var Handled: Boolean ) of object;

C++Builder

__property TCodeSiteSendMsgEvent OnSendMsg

 

typedef void __fastcall (__closure *TCodeSiteSendMsgEvent)(System::TObject* Sender, const TCodeSiteMsgData &MsgData, bool &Handled);

Remarks

Write an event handler for the OnSendMsg event to "hook" into the sending process of CodeSite messages.  The event handler is passed three parameters.  The Sender represents the CodeSite object that is being used to send the message.  The MsgData parameter is of type TCodeSiteMsgData, and represents an object that describes all attributes of the formatted message.  The Handled parameter controls whether or not the message is sent to the current destination.  Set Handled to True to prevent the message from being sent to the current destination.

If this event is handled and the Handled parameter is set to True, the CodeSite Dispatcher is not needed because the messages will never actually be sent anywhere.  A developer may choose to handle the OnSendMsg event to redirect CodeSite messages to a table in a database for example.
