Remarks

Use these methods to send a CodeSite message to the current destination.  The overloaded versions accept different data types in their native formats.  This eliminates the need to convert the data in your code before sending it to CodeSite.  For .NET development, this also eliminates the need to box and unbox the data to be logged.

NOTE

Bitmap images can be quite large.  As a result, sending large bitmaps over the network to a remote destination is not recommended.

Example

CodeSite->Send( "Welcome to CodeSite" );          // Simple text message

CodeSite->Send( csmGreen, "Start Demo" );         // Simple text with highlighted icon

CodeSite->Send( "FrmMain->Height", Height );      // Integer value

CodeSite->Send( "Main Form", this );              // Object properties

CodeSite->Send( "Songs", listBox1->Items );       // String list

CodeSite->Send( "Portrait", Image1->Picture );    // Bitmap image

 

C++Builder NOTE

Due to differences in method resolution between Delphi and C++Builder, there are certain situations in C++Builder where a Send method call may result in a compiler error describing an ambiguity between two or more Send methods. For example, the following statement will result in a compiler error:
 
CodeSite->Send( csmGreen, "Sample Message" );
 
The ambiguity results from how C++Builder handles string parameters and BOOL parameters.  Typecasting the affected parameter will usually eliminate the compiler error.  For example,
 
CodeSite->Send( csmGreen, (UnicodeString) "Sample Message" );
