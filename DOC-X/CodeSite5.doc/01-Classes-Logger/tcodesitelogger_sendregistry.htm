Remarks

Use this method to evaluate all of the name-value pairs that exist in the specified Registry key and then send the results to the current destination.  The RootKey parameter is one of the standard root Registry keys such as HKEY_LOCAL_MACHINE or HKEY_CURRENT_USER.  The Key parameter represents the path to the desired Registry entries.

If ProcessSubKeys is False, only the name-value pairs that exist in the specified key are evaluated.  Otherwise, all sub-keys and their name-value pairs are evaluated.

The resulting data is formatted like an INI file, with each key being represented as an INI file section.  Within the section, the name-value pairs are listed in typical "name"="value" format.

The first version of this method assumes the HKEY_CURRENT_USER root key.

C++Builder Users

In C++Builder 2009, new applications has NO_STRICT defined by default. This conditional symbol interferes with certain Delphi translated methods. This conditional symbol causes the compiler to report an unresolved external symbol error for the SendRegistry method. To resolve the error, simply remove the NO_STRICT conditional symbol.
 
For more information about NO_STRICT you can read the Known Problems section in the RAD Studio 2009 Release Notes. http://support.codegear.com/el/article/38475
