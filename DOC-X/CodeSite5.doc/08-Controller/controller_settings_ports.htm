CodeSite Controller

Settings: Ports

    

In order for the Dispatcher to receive CodeSite messages sent to it via TCP (or UDP), the Dispatcher must be monitoring the appropriate ports.  Be sure the port numbers match the port numbers being used to send the CodeSite messages.

NOTE

If you are running a personal firewall on your system such as Norton Internet Security, ZoneAlarm, or Windows Defender, your firewall software will ask you to confirm that you want to allow the CodeSite Dispatcher access to the network and to act as a server.

The port numbers may need to be changed to get through a network firewall. Please ask your network administrator if there are restrictions on the default port numbers.  The default port numbers used by the CodeSite Dispatcher are 3434 for CodeSite messages sent using TCP and 3435 for CodeSite messages sent using UDP.  Click the Default Ports button to quickly restore the default values.
