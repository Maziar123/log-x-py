Sending Custom Data: ICodeSiteCustomData (Delphi)

    

Note: *Not Available in CodeSite Express

The following example code illustrates how to implement the ICodeSiteCustomData interface in Delphi.  The first step is to define your custom class that implements the ICodeSiteCustomData interface. If you are simply interested in implementing the ICodeSiteCustomData interface, but do not require the full-blown interface management requirements of reference counting, etc., then you can descend your base class from TCodeSiteInterfacedObject instead of TInterfacedObject.

TCodeSiteInterfacedObject does NOT manage a reference count and therefore, will not get automatically destroyed when the object goes out of scope.

IMPORTANT

If you do descend your class from TInterfacedObject *and* you create instances of the class directly rather than interface references, then you will need to set the object reference to nil before freeing the object.  This is not necessary when descending from TCodeSiteInterfacedObject.

type

  TBaseballPlayer = class( TCodeSiteInterfacedObject, ICodeSiteCustomData )

  protected

    procedure GetFormattedData( Data: TStringList );

    function InspectorType: TCodeSiteInspectorType;

    function MsgType: Integer;

    function TypeName: string;

  public

    LastName: string;

    FirstName: string;

    Position: string;

    Salary: Currency;

    Games: Integer;

    AtBats: Integer;

    Hits: Integer;

    ERA: Single;

    AllStar: Boolean;

  end;

 

procedure TBaseballPlayer.GetFormattedData(Data: TStringList);

begin

  // The following code formats the player data in a text format

 

  Data.Add( FirstName + ' ' + LastName );

  if AllStar then

    Data.Add( 'All-Star ' + Position )

  else

    Data.Add( Position );

  Data.Add( '' );

  Data.Add( 'Salary: ' + Format( '%m', [ Salary ] ) );

  Data.Add( '' );

  Data.Add( 'Games Played: ' + IntToStr( Games ) );

 

  if Position = 'Pitcher' then

    Data.Add( 'ERA: ' + FormatFloat( '0.00', ERA ) )

  else

    Data.Add( 'Batting Avg: ' + FormatFloat( '0.000', Hits / AtBats ) );

 

  // The following code can be used to format the player data in 

  // a name-value property grid

  (*

  Data.Add( 'LastName=' + LastName );

  Data.Add( 'FirstName=' + FirstName );

  Data.Add( 'Position=' + Position );

  Data.Add( 'Salary=' + Format( '%m', [ Salary ] ) );

  Data.Add( 'Games=' + IntToStr( Games ) );

  Data.Add( 'AtBats=' + IntToStr( AtBats ) );

  Data.Add( 'Hits=' + IntToStr( Hits ) );

  Data.Add( 'ERA=' + FormatFloat( '0.00', ERA ) );

  if AllStar then

    Data.Add( 'AllStar=True' )

  else

    Data.Add( 'AllStar=False' );

  *)

end;

 

function TBaseballPlayer.InspectorType: TCodeSiteInspectorType;

begin

  Result := itStockText;

end;

 

 

function TBaseballPlayer.MsgType: Integer;

begin

  Result := 0;

end;

 

 

function TBaseballPlayer.TypeName: string;

begin

  Result := 'TBaseballPlayer';

end;

 

The above TBaseballPlayer class could then be used as the following method demonstrates.

IMPORTANT

Please note the comments in the source code below

 

procedure TForm1.Button1Click(Sender: TObject);

var

  P: TBaseballPlayer;

begin

  P := TBaseballPlayer.Create;

 

  P.LastName := 'Starlin';

  P.FirstName := 'Castro';

  P.Position := 'Short Stop';

  P.Salary := 2400000.00;

  P.Games := 125;

  P.AtBats := 800;

  P.Hits := 221;

  P.ERA := 0;

  P.AllStar := True;

 

 

  // This version is the same as sending an object's properties by

  // scanning the RTTI. In this example, no RTTI is available. 

  CodeSite.Send( 'Player Data', P );

 

  // This version treats P as ICodeSiteCustomData and the 

  // GetFormmatedData method will be called.

  CodeSite.Send( 'Player Data', P as ICodeSiteCustomData );

 

  // The above statement is the same as the following:

  CodeSite.SendCustomData( 'Player Data', P as ICodeSiteCustomData );

 

 

  {$IFDEF USING_TINTERFACEDOBJECT}

  // IMPORTANT WIN32 NOTE

  // If you descend your custom class from TInterfacedObject and

  // you write code that deals with instances of the class rather

  // than interface references, then you will need to release the

  // interface references that get allocated when you call the 

  // CodeSite.Send methods.  To do this, simply set the object reference

  // to nil before calling Free.

  //

  // PLEASE NOTE that this step is not necessary if you descend 

  // your class from TCodeSiteInterfacedObject.

  P := nil;

  {$ENDIF}

 

  P.Free;

end;
