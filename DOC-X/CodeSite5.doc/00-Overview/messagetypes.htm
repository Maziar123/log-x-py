Message Types & Content Values

    

Each CodeSite message has a MsgType property and a MsgContent property. The MsgType is used to highlight or emphasize a message (e.g. an Error versus a Reminder).  The MsgContent is used to identify the content of the message (e.g. Text versus Object) and also controls which inspector is used to display the additional content.  Each property value has an associated icon for easy identification.

The following tables lists the CodeSite Message Types. These values can be used in any method that accepts a MsgType parameter.

The following table lists the CodeSite Message Content values. These values indicate that the message contains additional details that can be displayed in the Viewer's Inspector Pane. Each content value is associated with a specific inspector. Inspector message types are automatically used by the T|CodeSiteLogger methods where necessary and are rarely used directly in logging statements. These values can be used in any method that accepts a MsgContent parameter.

Message Contents

Text

Simple text based message with no additional content.

StringList

The contents of the specified list is attached to the message.

Object

A list of all published properties and their current values for the specified object is attached to the message.

ComponentAsText

(Delphi/C++Builder) A text representation of the streaming format for the specified component is attached to the message.

Color

The RGB and HSL values along with a sample of the specified color are attached to the message.

Bitmap

The specified bitmap is attached to the message.

TextFile

The contents of the specified text file is attached to the message.

Memory

A hexadecimal view of the specified memory block is attached to the message.

MemoryStatus

A list of system memory statistics is attached to the message.

Window

The system details and styles for the specified window handle are attached to the message.

Registry

The contents of the specified registry key are attached to the message.

SystemInfo

A list of information about the system (i.e. OS version, computer name, user name, screen resolution, etc.) is attached to the message.

VersionInfo

The details of the application's version information block is attached to the message.

XmlData

Xml data is attached to the message.

Collection

The properties of each item from the specified collection are attached to the message.

StackTrace

(.NET Only) The current stack trace is attached to the message.

CustomText

The contents of a custom data element formatted as text is attached to the message.

CustomProperties

The contents of a custom data element formatted in a name-value property grid is attached to the message.

CustomTable

The contents of a custom data element formatted in a multi-row, multi-column table is attached to the message.

CustomHierarchy

The contents of a custom data element formatted in a hierarchical tree is attached to the message.

Custom

Base message type used in defining custom message inspector types.

NOTE

Message Type values and Message Content values are defined as constants is all supported frameworks.  To reference any of the constants in your code, you will need to prefix the constant name. In Delphi, the constants are prefixed with "csm" (eg. csmGreen or csmCollection). In .NET, the constants are prefixed with "CodeSiteMsgType." (eg. CodeSiteMsgType.Level4 or CodeSiteMsgType.Object).
