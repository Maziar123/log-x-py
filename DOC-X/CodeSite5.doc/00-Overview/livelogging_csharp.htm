Getting Started

Live Logging Tutorial: C#

    

1.Start Visual Studio if it is not already running.  There is no need to start the CodeSite Dispatcher or the CodeSite Live Viewer as they will be started automatically when we run our sample program.

2.Create a new C# Windows Forms Application. Next, add the Raize.CodeSiteLogging.dll assembly to the References section of the new project.  (Add the Raize.CodeSiteExpress.dll assembly if you are using CodeSite Express.) The assembly is located in the Raize\CS5\Deploy\DotNET directory.

3.Next, add the Raize.CodeSiteLogging namespace in a using statement at the top of the form file.
 
   using Raize.CodeSiteLogging;

4.Drop a button on the form, double-click it, and in the Click event handler write the following:
 
   CodeSite.Send( "My first message" );

5.Compile and run the application.  When the program is up and running, click on the button.  The CodeSite Dispatcher will be started followed by the CodeSite Live Viewer, and in the message log you will see your first CodeSite message.

6.Next, stop the sample application and add the following code to the Click event handler:
 
   CodeSite.Send( "Main Form", this );

7.Recompile and run the application. Each time you click the button, you will see two messages appear in the CodeSite Viewer.  The icons displayed next to each message indicate the message type and the message content.  The icon indicates that the "Form1" message has an object attachment.

NOTE

If the messages do not show up in the viewer, make sure that either the TRACE or CodeSite conditional symbol is defined for the project. If neither of these conditional symbols is defined, the CodeSite methods are not generated when the project is compiled.

8.To view the attached object information, make sure the button is depressed so that the Inspector pane is visible and then select the "Form1" message.  The public properties for your form will be listed in the inspector pane.

9.Again, stop the sample application.  Next, update the Click event handler so that your method looks like the following:
 
   CodeSite.EnterMethod( "button1_Click" );
   CodeSite.Send( "My first message" );
   CodeSite.Send( "Main Form", this );
   CodeSite.ExitMethod( "button1_Click" );
 

10.Recompile and run the application.  This time, when you click the button, you will see some structure to the messages sent to the CodeSite Live Viewer.  Note how the "My first message" and "Form1" messages are indented in between the and messages.
 
By adding EnterMethod and ExitMethod calls to various methods of your application, you can generate a log that describes which methods were executed and when.

 

This concludes the C# Tutorial.  Please return to the Live Logging Tutorial to continue.
