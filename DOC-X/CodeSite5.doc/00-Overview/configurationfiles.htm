Advanced Techniques - CodeSite and .NET

.NET Configuration Files

    

Note: *Not Available in CodeSite Express

CodeSite supports storing the details of your destinations in the application configuration file. This greatly enhances the flexibility you have when sending messages from your application to any CodeSite destination. A change in a destination, e.g. a server name changed where you are sending messages to, does not result in having to recompile your application, but merely a change in the configuration file.

Register the Configuration Handler

The first step in this process is to register the CodeSite configuration handler. This is done by adding a section in your application configuration file, like this:

<?xml version="1.0" encoding="utf-8" ?>

<configuration>

 

  <!-- Register our Handler with the system -->

  <configSections>

    <section name="CodeSite" 

             type="Raize.CodeSiteLogging.CodeSiteConfiguration, Raize.CodeSiteLogging" />

  </configSections>

 

The CodeSite Section

This allows you to specify all the configurations in a CodeSite section of your config file. The layout of the CodeSite section is as follows:

  <CodeSite>

 

    <Manager enabled="true" defaultDestination="Debug" />

 

    <!-- Only have one Manager node specified at a time

    <Manager enabled="true" defaultDestination="Debug">

      <Connection type="CopyData"/>

    </Manager>

 

    <Manager enabled="true" defaultDestination="Debug">

      <Connection type="TCP" host="helpdesk" port="2222"/>

    </Manager>

    -->

 

    <Destination name="Release">

      <LogFile filePath="$(MyDocs)" fileName="Sample.csl" maxParts=10 logByDate="true" logByDateFormat="yyyy-mm-dd"/>

    </Destination>

  

    <Destination name="Debug">

      <Viewer/>

      <LogFile filePath="C:\Logs\" fileName="Sample.csl"/>

    </Destination>      

 

    <Destination name="Full">

      <Viewer/>

      <LogFile filePath="C:\Logs\" fileName="Sample.csl" maxSize="256"/>

      <TCP host="tahu"/>

    </Destination>      

    

    <Destination name="Remote">

      <TCP host="tahu" port="1111" remoteDestinationString="Viewer"/>

    </Destination>

    

  </CodeSite>

   

</configuration>

 

The Manager Section

The CodeSiteManager defines a DefaultDestination property, which is set for the CodeSite Live Viewer by default. This default can be overridden in the config file as shown above. That is, the Manager element specifies that it will start Enabled and that the DefaultDestination value is the one specified in the “Debug” destination.

Once you have setup a config block, you can use the CodeSiteManager to quickly access the settings defined in the config file.  For example,

CodeSite.Destination = CodeSiteManager.Destinations[ "Release" ];
