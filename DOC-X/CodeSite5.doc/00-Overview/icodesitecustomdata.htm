ICodeSiteCustomData Interface

    

Interface definition for formatting custom data so that it can be displayed by the CodeSite viewers.

Remarks

The easiest way to format your custom data for use in CodeSite is to have your custom class implement the ICodeSiteCustomData interface. ICodeSiteCustomData defines four methods: GetFormattedData, InspectorType, MsgType, and TypeName.

GetFormattedData is where you will actually format your custom data as desired.  Specifically, developers will populate a TStringList in Delphi/C++Builder, and a StringBuilder in .NET.

The InspectorType method is used to indicate what type of inspector the CodeSite Viewer should use to view your custom format.  The choices are StockText, StockProperties, StockTable, StockHierarchy, and Custom.

If you are using one of the stock inspectors to display your custom formatted data, then for the MsgType method, you can simply return zero.  If you are specifying a Custom inspector to be used, then you need to return an integer value that will be used to bind your formatted messages data to the desired custom inspector.  If your custom data is for your own personal use or for use within your department or organization, then you should define your integer constant based on the User message type constant plus some offset.  The valid offset range is 1 to 32,000.

If you are creating the class and custom format for public use outside of your department or organization, then you should define your constants based on the Public message type constant plus some offset.  Please contact Raize Software (support@raize.com) to obtain a valid offset that does not conflict with other public formatters.

Please read the topic on Sending Custom Data for more information.
