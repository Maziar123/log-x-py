Deploying CodeSite

    

The CodeSite Logging System is an excellent tool for tracking down problems in your code, especially during the development process. However, continuing to log CodeSite messages after an application is deployed to a client machine is equally valuable. In fact, problems that arise on a client machine are often much more difficult to track down, especially when the problem cannot be recreated on your development system!

Both Live Logging and File Logging sessions may be conducted on a client's computer. Regardless of the logging style, the CodeSite Dispatcher must be accessible. That is, the Dispatcher must be present on the client's computer, or the CodeSite Dispatcher must be running on another system that is connected to the client's computer. If you wish to conduct a live logging session, then the CodeSite Live Viewer must also be available.

Fortunately, as part of the CodeSite Tools, the Dispatcher and Live Viewer are freely distributable with your application.  The recommended method for deploying the CodeSite Tools is to use the CS5_Tools.exe installation program that is provided in the CS5\Tools directory.  The CS5_Tools.exe installation program can even be run silently so that it can be easily incorporated into the primary installation process of your own applications.  Please read the CS5\Tools\CodeSiteToolsSetup.txt file for details on customizing the installation process.

TIP

If you are only interested in generating a log file for your application, and you do not require the Live/File Viewer nor the Controller, then an alternative to installing the CodeSite Tools is to simply copy the CSDispatcher.exe file to the same directory as your application. When your application tries to send its first CodeSite message, the CSDispatcher.exe program will be started and the log file will be updated by the Dispatcher.  When your application closes, you may wish to call the T|CodeSiteLogger.CloseDispatcher method to terminate the Dispatcher.
