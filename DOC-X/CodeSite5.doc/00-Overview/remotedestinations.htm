Advanced Techniques

Remote Destinations

    

There are two basic destinations for CodeSite messages: a CodeSite Live Viewer; and a CodeSite Log File. However, these two destination types may reside either locally or on a remote machine. The CodeSiteManager.DefaultDestination and T|CodeSiteLogger.Destination properties are used to specify where (and how) CodeSite messages are routed to their final destination after they are received by a CodeSite Dispatcher.  As you can see from the CodeSite Architecture diagram, a CodeSite Dispatcher can direct messages directly to an instance of the Live Viewer, a log file, or to another instance of the CodeSite Dispatcher running on a different system. The Remote destination sub-types of T|CodeSiteDestination (i.e. TCP, UDP, HTTP) are used to establish Dispatcher to Dispatcher communication.

Setup

Both computers need to be connected to the same network.  Also make sure that TCP/IP support is installed on both computers.  On the local machine, create an application that sends some CodeSite messages.  Modify the Destination property of the T|CodeSiteLogger instances you are using (or the CodeSiteManager.DefaultDestination) to specify one of the remote destination types (TCP or UDP)--HTTP is covered later in this topic.

The CodeSite Dispatcher must be running on the remote computer. Please read the Deploying CodeSite topic for details on deploying the Dispatcher and other CodeSite Tools. You also need to make sure the remote Dispatcher is monitoring the appropriate ports for remote CodeSite messages.  This option is located in the Dispatcher Settings dialog on the Ports page.  Likewise, you can configure this option during the installation of the CodeSite Tools.

Next, run your application on the local computer and try to send some CodeSite messages.

Troubleshooting Checklist

If your CodeSite messages do not appear on the remote computer, use the following check list to help locate the problem:

1.Make sure the CodeSite Dispatcher is running on the remote computer.

2.From both computers, make sure you are able to ping the other computer using both the IP address and HostName?

3.If you are using a firewall, make sure the ports you are using are open.  The default port for TCP is 3434 and the default port for UDP is 3435.

4.If the ports are blocked, ask your network administrator if they can be opened or request a valid port number to use.  The port numbers can be changed in the CodeSite Dispatcher Settings dialog.

IMPORTANT

Please note that if you change the port numbers, then you must alter the Destination property of your T|CodeSiteLogger instances (or CodeSiteManager.DefaultDestination) settings to use the new port number.

5.If you are trying to send messages to the CodeSite Live Viewer running on the remote computer, make sure the Live Viewer is available on the remote computer and not just the Dispatcher.

6.On the local computer, check the Dispatcher Log for any error or warning messages, and make sure the appropriate destination type was registered.

7.On the remote computer, also check the Dispatcher Log for any error or warning messages, and make sure the appropriate receiver was started.

HTTP Destinations and the CodeSite Web Modules

CodeSite also supports sending messages to a remote computer via a web server (see CodeSite Architecture for a diagram of the message path).  More precisely, the CodeSite Dispatcher can use HTTP to send a CodeSite message to the CSCgiDispatcher.exe or CSIsapiDispatcher.dll web modules running on a web server.  The web modules simply redirect the message to another CodeSite Dispatcher using TCP.

In the CS5\Bin\HttpUtil folder, you will find the CSCgiDispatcher.exe and the CSIsapiDispatcher.dll web modules.  In order to send CodeSite messages to a remote computer via a web server, you will need to copy one of these two modules to your web server.

Next, please read the HTTP Destination Type topic for detailed information on how to set the Destination property of your T|CodeSiteLogger instances appropriately and how to create the appropriate INI file sections that instruct the web module where to forward the messages it receives.

After setting the Destination property, copying the web module to the web server, and creating the appropriate INI file sections, you can attempt to send CodeSite messages to the remote computer via the web server.

IMPORTANT

Please note that the CodeSite Dispatcher MUST be running on the Remote HOST machine (referenced in the Host value in the INI file) in order to receive the messages being routed by the web module.  Also note that by using the aliases to route messages to another computer, it is not necessary to have the CodeSite Dispatcher running on the web server.
