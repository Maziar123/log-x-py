CodeSite Logging System

Advanced Techniques

    

The topics in this section describe some of the more advanced features available in the CodeSite Logging System.

Sending Custom Data

Although the methods provided in the T|CodeSiteLogger class will handle most of the data types used in your applications, there may be times when you want to send data of a type that the logging class does not support.  For example, if you are using Delphi or C++Builder, you may want to send the field values of a record structure, or perhaps the properties of an object whose class type does not include runtime type information (RTTI).  Regardless of the framework that you are using, you still may wish to format your data differently than the built-in methods do. For example, a logical interpretation of several different data structures.

Remote Destinations

All CodeSite messages ultimately get routed to either the CodeSite Live Viewer or to a CodeSite Log File.  However, CodeSite allows you to route your messages to a log file or the Viewer running on a remote computer.  This topic describes how to get started sending CodeSite messages to remote computers.

Destination Strings

The T|CodeSiteDestination class provides an easy programmatic way of manipulating the destinations for CodeSite messages.  However, the underlying messaging system used by CodeSite (namely the CodeSite Dispatcher) uses destination strings, which is a text-based representation of the properties that define a destination. Destination Strings are also used in specifying remote destinations.

CodeSite and .NET

The CodeSite Logging System provides support for the Microsoft .NET Framework 2.0 and later. The CodeSite Logging Classes are accessible regardless of the development tool or language you are using. Furthermore, CodeSite provides some additional features specifically designed for developers using the .NET Framework.
