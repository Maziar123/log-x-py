File Logging Tutorial

    

Sending CodeSite messages to a log file is no different than sending messages to the CodeSite Live Viewer.  You make the same method calls to send the actual data.  However, before you starting sending your messages, you must instruct the CodeSite Dispatcher that messages received from your T|CodeSiteLogger instances need to go to a different destination.  This is accomplished by using the T|CodeSiteLogger.Destination, or if you are using multiple logger instances, the CodeSiteManager.DefaultDestination property.

The following example builds on the sample presented in the Live Logging Tutorial, and illustrates how to perform file logging.

Please note that your CodeSite messages will still be routed through the CodeSite Dispatcher--see CodeSite Architecture for more details.  As a result, if you wish to send CodeSite messages from your application while it is running on a client's computer, you will need to install the CodeSite Dispatcher on that client's computer.  Please read the Deploying CodeSite topic for more details.
