Getting Started

Live Logging Tutorial: C++Builder

    

1.Start Delphi if it is not arleady running.  There is no need to start the CodeSite Dispatcher or the CodeSite Viewer as they will be started automatically when we run our sample program.

2.Create a new application and then drop a button on the form.

3.Next, you need to add a #pragma statement to the Unit1.cpp file. The statement instructs the complier/linker to include the CodeSiteLoggingPkg library (or the CodeSiteExpressPkg library if you are using CodeSite Express).
 
   // Add this line if using CodeSite Studio in a 32-bit application
   #pragma comment(lib,"CodeSiteLoggingPkg.lib")
                                                 
   // Add this line if using CodeSite Express in a 32-bit application
   #pragma comment(lib,"CodeSiteExpressPkg.lib")
 
 
   // Add this line if using CodeSite Studio in a 64-bit application
   #pragma comment(lib,"CodeSiteLoggingPkg.a")
 
   // Add this line if using CodeSite Express in a 64-bit application
   #pragam comment(lib,"CodeSiteExpressPkg.a")
 

4.Next, you need to add another #pragma statement to the Unit1.cpp file. This statement links in the CodeSiteLogging unit, which contains the CodeSite Logging Classes.
 
   #pragma link "CodeSiteLogging"

5.Likewise, the Unit1.hpp header file needs to include the CodeSiteLogging.hpp file:
 
   #include "CodeSiteLogging.hpp"

6.Double click the button and in the OnClick event handler for the button write the following:
 
   CodeSite->Send( "My first message" );

7.Compile the application. (If you receive an error that CodeSiteLogging.hpp cannot be found, make sure that the appropriate Raize\CS5\Lib\RSnnnn directory in listed in the project's Include Path and Library Path.)

8.Next, run the application.  When the program is up and running, click on the button.  The CodeSite Dispatcher will be started followed by the CodeSite Live Viewer, and in the message log you will see your first CodeSite message.

9.Next, stop the sample application and add the following code to the OnClick event handler:
 
   CodeSite->Send( "Main Form", this );

10.Recompile and run the application. Each time you click the button, you will see two messages appear in the CodeSite Live Viewer.  The icons displayed next to each message indicate the message type and the message content.  The icon indicates that the "Form1" message has an object attachment.

11.To view the attached object information, make sure the button is depressed so that the Inspector pane is visible and then select the "Form1" message.  The published properties for your form will be listed in the inspector pane.

12.Again, stop the sample application.  Next, update the OnClick event handler so that your method looks like the following depending on the CodeSite Edition your are using:
 
CodeSite Studio
 
   CodeSite->TraceMethod( "Button1Click" );
   CodeSite->Send( "My first message" );
   CodeSite->Send( "Main Form", this );
 
CodeSite Express
 
   CodeSite->EnterMethod( "Button1Click" );
   CodeSite->Send( "My first message" );
   CodeSite->Send( "Main Form", this );
   CodeSite->ExitMethod( "Button1Click" );
 

13.Recompile and run the application.  This time, when you click the button, you will see some structure to the messages sent to the CodeSite Viewer.  Note how the "My first message" and "Form1" messages are indented in between the and messages.
 
By adding TraceMethod calls (or EnterMethod and ExitMethod calls) to various methods of your application, you can generate a log that describes which methods were executed and when.

 

This concludes the C++Builder Tutorial.  Please return to the Live Logging Tutorial to continue.
