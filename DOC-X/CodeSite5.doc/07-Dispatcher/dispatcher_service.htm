CodeSite Dispatcher

Running as a Service

    

The CodeSite Dispatcher can be run as a regular application or as service.  In order to run the Dispatcher as a service, you must first install the Dispatcher into the Service Control Manager (SCM).  To do this, first make sure that the Dispatcher is not already running, then simply run the CSDispatcher.exe program and specify the /install option as shown below:

CSDispatcher /install

Once the Dispatcher is installed as a service, you can start the Dispatcher Service using the Services Control Panel applet, or you can type the following at a command prompt.  Please note that there is no space in CodeSiteDispatcher.

net start CodeSiteDispatcher

By default, when the Dispatcher is installed as a service it's Startup Mode is set to Automatic, which means that the Dispatcher will be automatically started when the system is started.

When running as a service, the Stop Dispatcher command is not available. To stop the Dispatcher service, use the Services Control Panel applet, or type the following at a command prompt:

net stop CodeSiteDispatcher

To remove the Dispatcher from the Service Control Manager, first stop the service and then run the CSDispatcher.exe program and specify the /uninstall option as shown below:

CSDispatcher /uninstall

NOTE

When the Dispatcher service is started you will usually see the icon appear in the system tray. However, it is possible that the Dispatcher  icon may not be visible. This can happen on systems that allow multiple users to be logged into the system at the same time. For example, Windows XP with Fast User Switching. In situations like this, only the first logged on user will see the Dispatcher icon in the system tray.

NOTE

By default, the CodeSite Dispatcher service is run from the LocalSystem account. The Dispatcher service can be run from a different account, but if you wish to perform live logging on the local machine, then LocalSystem must be used, because LocalSystem is the only account that is allowed to interact with the desktop, which is required for communicating with the Live Viewer on the local machine.

Please note that this is only required for sending messages to the CodeSite Live Viewer directly on the local machine.  If you wish to send your messages to a log file or to a remote Live Viewer, then the Dispatcher can be logged on as another user and does NOT need to interact with the Desktop. The only requirement in this case is that your T|CodeSiteLogger instances must connect to the Dispatcher service using TCP. Use the CodeSiteManager.ConnectUsingTcp method or the T|CodeSiteLogger.ConnectUsingTcp method.
